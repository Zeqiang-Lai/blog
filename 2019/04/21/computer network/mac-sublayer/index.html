Medium Access Control Sublayer

Posted on 2019-04-21 Edited on 2019-04-22 In 计算机网络

Medium Access Control Sublayer 也叫 MAC 子层。

MAC子层的解决的问题存在于：利用广播(Broadcast)的网络传输。点对点的传输不需要这个子层。

具体来说，MAC子层的作用是合理分配单一的网络资源给多个用户。这里的网络资源可以是多个用户共享的无线网的一个频段，也可以是多个用户共享的一条网线。

在计算机网络的层次结构中，MAC子层为数据链路层的一部分，并在其底部。

1 Motivation

MAC子层源自”有限的网络资源的分配”问题。

传统的解决资源分配问题的方案是静态分配。

1.1 Static Channel Allocation

静态分配使用

  • FDM(Frequency Division Multiplexing): 将带宽(bandwidth)分成n份，平均分给各个用户。
  • 或TDM(Time Division Multiplexing): 用户分时的使用网络资源（简单的说，一人用一小会，然后轮流用）。

但这样的问题是，网络资源利用率低，效率也低。Why？

  1. 用户的网络使用量不是平均分布的，用户没用的时候，分配给他的带宽(FDM)或时间片(TDM)会被浪费。 -> 利用率低
  2. 当用户需要用的时候，由于只分配到少量的带宽，或时间片，网速情况不容乐观。-> 效率低下。

1.2 Dynamic Channel Allocation

静态分配不好，那就动态分配呗。

什么是动态分配？

动态分配：动态分配网络资源给需要的用户。例如，现在只有一个用户请求网络资源，那就把资源全部给他。要是有两个，那就平分。怎么分是后面讲的协议要解决的问题，不一定就是平分。

2 Multiple Access Protocols

前面说了”怎么动态分配”要后面讲，现在的Multiple Access Protocols就是要回答这个问题。通过这些协议，限定各个网络用户之间使用网络的方法，进而实现动态分配。

2.1 ALOHA

ALOHA是夏威夷语，什么含义不重要，不过这个方法是在夏威夷大学提出来的。

ALOHA协议有两个变种：

  1. pure：原始版本。
  2. slotted：分时版本。

2.2 Carrier Sense Multiple Access Protocols

前面的ALOHA，不管是Slotted还是Pure都不管其他人是否在使用网络，然而，在LAN中，我们是有办法去监测其他人当前的行为的。通过监测其他人目前的情况，我们可以合理的调整自己的行为，进而获得更高的信道利用率。

下面都的方法都是基于上述出发点的。

2.2.1 Persistent and Nonpersistent CSMA

1-persistent CSMA：发送之前先监测有没有别人在用，如果有，等着。一旦信道空闲，立刻开始发送（1的含义表示，一旦空闲以100%的概率开始发送）。

问题1：仍然很容易发生冲突，因为如果两个用户在第三个用户发送的时候都准备好，那么他们会一起等用户3完成，然后等3完成后，一起发送，必定冲突，而这种情况将会很常见。

问题2：propagation delay

  • 补下propagation delay的问题

nonpersistent CSMA：也是要用之前先看别人有没有在用，但是如果有的话，不一直监测，而是随机等一段时间，再看信道能不能用。

信道利用率比1-persistent要高，但是延迟也更高。

p-persistent CSMA：该方法以时间片的方法使用信道，然后以p的概率使用当前时间片。
