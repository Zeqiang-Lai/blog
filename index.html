LR(0), SLR(1), LR(1), LALR(1)分析法比较

Posted on 2019-06-22 Edited on 2019-06-25 In 编译原理

四者都是自低向下的分析方法。

四种方法十分类似，后者是前者的改进，在识别语言的能力上有所加强。

  1. LR(0)是基础。
  2. SLR(1)根据follow集合解决冲突问题。
  3. LR(1)根据当前活前缀的实际follow集合解决冲突问题。
  4. LALR通过合并LR(1)中的同心项目集减少状态数。

LR(0)

LR(0)分析法需要一个分析表，一个状态栈，一个符号栈。

  1. 分析表指明了根据当前状态和当前符号的操作以及下一个状态。
  2. 符号栈存储当前已经分析出来的部分。

分析表的格式如下所示：

state action goto
a b # S
0 s3 acc 4
1 r2

分析过程

初始状态为0，分析开始时，状态栈中为[0]。

  • 这时候从待分析字符串中压入一个符号进符号栈。
    • 根据分析表的action 部分，决定执行哪种操作。
    • s[i]表示shift操作。虽然这里说到shift，但在这里我们并不会实际移动什么。这里的操作为往状态栈中压入状态i。
    • r[i]表示reduce操作，它表示当前符号栈中有可规约的子串，并且我们应该使用第i个产生式进行规约。因此，这里的操作是
      • 根据产生式i弹出对应数目的符号，并将规约后的符号压入栈中。
      • 接着，弹出状态栈的栈顶符号，根据新的栈顶状态，还有刚刚规约出来的符号，查分析表中的goto表，压入一个新的状态进状态栈。
    • 若表中对应项为空，表示出错。
  • 重复压入符号，直到无符号可压或分析完毕为止。

分析表的构造

先给出item的定义:

An LR(0) item of a grammar G is a production of G with a dot at some position of the body.

  • 根据起始item，计算item的closure。
  • 每个closure对应一个状态
  • 然后根据不同的输入字符，求这个closure下一个状态（新的一个closure），和当前要执行的动作，s(shift)还是r(reduce)。
  • GOTO表指示了规约完的下一个状态是哪个状态。

每个closure也叫做Canonical item set，只需要保存kernel item，non kernel item可以从kernel item中推导出来。 -> 压缩parsing table的基础。

SLR(1)

LR(0)分析表构造完毕之后，可能会发现一个项目集中存在移进规约冲突。

SLR(1)的解决方案是看冲突项目左侧符号的follow集合，若二者不交叉，则可以根据下一个字符解决冲突。

若二者交叉，则SLR(1)不能解决。

LR(1)

SLR(1)的follow集合中有很多元素，但并不是每个元素都可能是其对应项目实际的follow字符。

LR(1)根据推导出当前项目的前一个项目，记录当前项目的实际follow集合。LR(1)称follow集合中的字符为搜索符。

类似的，根据下一个字符，如果能区分冲突项目，则LR(1)能够解决冲突，否则，LR(1)无法解决。

LALR(1)

LR(1)分析表的构造引入了很多新的状态，有些状态（项目集）中的项目实际上只是搜索符不一样。LALR(1)分析法把这些项目集称为同心项目集。

如果将这些同心项目集合并，新得到的项目集规范族不存在冲突，则该文法称为LALR文法。

  • 值得注意的是，若发生冲突，只会产生归约-归约冲突，不会产生移进规约冲突。Why？

Transport Layer

Posted on 2019-06-06 Edited on 2019-06-13 In 计算机网络

How to prevent duplicate packet in transport layer?

The key point is to assign a unique number to each packet, just like datalink layer does.

  1. First, set up a maximum lifetime for each packet. Usually in hop counter.
  2. Then define a period time T, which is usually some small multiple of the true maximum packet lifetime.(in secs).
  3. Each packet in a period of time T will has a unique number.
  4. If there exists duplicated packets, they must have different number.

Duplicated packets can only exists in a period of time T after the first packet was sent, otherwise packets dead.

So if we can identify the same packets with different number, we can detect duplicated packets and reject them.

the technical term of number here is “sequence number”.

Three way handshake

  1. If host1 want to connect to host2, host1 must send a CR packet to host2 first.

  2. After receiving a CR packet, host2 choice a sequence number as the initial number, and send a packet back to host1.

  3. After receiving the ack from host2, host1 start to send packet with the intial packet that is specified by host2.

In this case,

  • if there is a duplicate CR, host1 can just reject the connection.

  • if there is a duplicate ack as well, host2 can still know this is a duplicate ack because the fact the host2 will only send ack with sequence number that is outside the forbidden region.

    • It means that there won’t be duplicate packet that have sequence number outside the forbiden region.
    • Then we can tell whether a packet is a duplicated one by determining whether it is in the forbidden region.

TCP Congesion Control

Network layer detect congestion when queues grow large at routers. Then network layer alarms transport layer to slow down the rate of traffic.

TCP use an ack clock to timing how long does it take for an ack to return to the sender.

AIMD

additive increase multiplicative decrease.

Slow Start

AIMD take too long to find an appropriate window size.

initil window <= 4 segment.

once receive an ack, send one more segment(2 segment) next time.

  • enforcing a minmum spaceing between data packets to send. use ack timer.

once slow start threshold is crossed, TCP switch from slow start to additive increase.

another improvement

assume three duplicative ack imply that a packet has been lost. then the packet can be retransmited.

Medium Access Control Sublayer

Posted on 2019-04-21 Edited on 2019-04-22 In 计算机网络

Medium Access Control Sublayer 也叫 MAC 子层。

MAC子层的解决的问题存在于：利用广播(Broadcast)的网络传输。点对点的传输不需要这个子层。

具体来说，MAC子层的作用是合理分配单一的网络资源给多个用户。这里的网络资源可以是多个用户共享的无线网的一个频段，也可以是多个用户共享的一条网线。

在计算机网络的层次结构中，MAC子层为数据链路层的一部分，并在其底部。
